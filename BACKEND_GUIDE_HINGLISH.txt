SKYLUXE BACKEND GUIDE - HINGLISH
================================

1. Project kis backend par bana hai?
Yeh project Django backend par bana hua hai.
Main entry file `manage.py` hai.
Project settings `skyluxe_site/settings.py` me hain.
Main project URLs `skyluxe_site/urls.py` me hain.


2. Backend ka overall structure
Project me backend 2 main apps me divide hai:

1. `pages`
   Is app ka kaam frontend website pages serve karna hai.
   Jaise home, about, contact, projects, clients aur lift detail pages.

2. `adminpanel`
   Is app ka kaam custom admin panel handle karna hai.
   Jaise admin login, dashboard, leads, products, projects, customers, services, profile, logout.


3. Request flow simple language me
Jab user browser me koi URL open karta hai:

1. Request pehle `skyluxe_site/urls.py` me aati hai.
2. Wahan se decide hota hai request `pages` app me jayegi ya `adminpanel` me.
3. Uske baad related `views.py` function chalta hai.
4. View data laata hai ya save karta hai.
5. Fir HTML template render hoti hai.


4. Main URL mapping
`skyluxe_site/urls.py` me ab ye main routing hai:

1. `''` -> `pages.urls`
   Matlab public website pages yahin se open hote hain.

2. `'admin/'` -> `adminpanel.urls`
   Matlab custom admin panel ka pura flow isi prefix ke andar hai.

3. `'django-admin/'`
   Yeh Django ka default admin panel hai.


5. Public website backend kaise kaam kar raha hai
File: `pages/views.py`

Yahan mostly simple render views hain:

1. `home()` -> `pages/home.html`
2. `about()` -> `pages/about.html`
3. `contact()` -> `pages/contact.html`
4. `projects()` -> `pages/projects.html`
5. `homelift()` -> `pages/home-lift.html`
6. `passengerlift()` -> `pages/passenger-lift.html`
7. `hospitallift()` -> `pages/hospital-lift.html`
8. `capsulelift()` -> `pages/capsule-lift.html`
9. `goodslift()` -> `pages/goods-lift.html`
10. `carlift()` -> `pages/car-lift.html`
11. `clients()` -> `pages/clients.html`

Matlab public website pages abhi dynamic content heavy nahi hain.
Backend in pages ko mostly sirf render kar raha hai.


6. Lead form backend kaise kaam karta hai
File: `pages/views.py`
Function: `save_lead`

Flow:

1. Jab frontend lead/contact type form submit hota hai to POST request `save_lead/` par aati hai.
2. Backend `request.POST` se fields leta hai:
   - name
   - email
   - phone
   - city
   - message
3. Phir `Lead.objects.create(...)` se database me record save hota hai.
4. Save hone ke baad user ko home page par redirect kar diya jata hai.

Important:
Abhi validation basic level ki hai.
Matlab strong form validation ya spam protection alag se nahi dikhti.


7. Lead model kya store karta hai
File: `pages/models.py`

`Lead` model me ye fields hain:

1. `name`
2. `email`
3. `phone`
4. `city`
5. `message`
6. `status`
   Default value: `New`
7. `is_read`
   Default value: `False`
8. `created_at`
   Auto timestamp

Matlab jab naya lead aata hai to:
`status = New`
aur
`is_read = False`


8. Admin login kaise kaam karta hai
File: `adminpanel/views.py`
Function: `admin_login`

Flow:

1. Admin login page URL:
   `admin/admin-login.html`

2. Agar user already authenticated hai to usko direct dashboard bhej diya jata hai.

3. Agar POST request aati hai:
   - username liya jata hai
   - password liya jata hai
   - `authenticate()` run hota hai

4. Agar credentials sahi hue:
   - `login(request, user)` run hota hai
   - user dashboard par redirect hota hai

5. Agar credentials galat hue:
   - same login page render hoti hai
   - error message dikhaya jata hai

Simple words me:
Custom admin login Django auth system use kar raha hai.


9. Dashboard kaise kaam karta hai
File: `adminpanel/views.py`
Function: `dashboard`

Dashboard login protected hai.
Decorator laga hai:
`@login_required(login_url='/dashboard/admin-login.html')`

Dashboard me yeh counts aate hain:

1. total leads
2. total products
3. total projects
4. total service requests
5. recent 5 leads

Matlab dashboard ek summary screen hai jahan admin ko project ka quick snapshot milta hai.


10. Leads section kaise kaam karta hai
File: `adminpanel/views.py`
Function: `leads`

Flow:

1. Jab admin leads page kholta hai to pehle `mark_leads_as_read()` call hota hai.
2. Yeh function sab unread leads ko read mark kar deta hai.
3. Phir saare leads latest first order me load hote hain.
4. Leads template me bhej diye jate hain.

Matlab admin leads open karte hi unread notification logically clear ho jati hai.


11. Admin context processor kya karta hai
File: `adminpanel/context_processors.py`

Yeh file smart kaam karti hai.
Har admin page par extra common data inject karti hai:

1. total leads count
2. unread/new leads count
3. logged-in user ka profile object

Isliye sidebar ya topbar me ye data baar-baar manually pass nahi karna padta.

16. Profile system kaise kaam karta hai
Model: `AdminProfile`

Isme admin user ke liye extra profile data store hota hai:

1. Django `User` ke sath OneToOne relation
2. profile_image

Special logic:
Agar profile image upload nahi hai to default avatar URL generate hota hai.
Matlab admin panel blank photo nahi dikhata.

Profile page par admin:
1. username change kar sakta hai
2. email change kar sakta hai
3. password change kar sakta hai
4. profile image upload kar sakta hai


17. Logout kaise kaam karta hai
Function: `admin_logout`

Flow simple hai:
1. `logout(request)` run hota hai
2. User ko admin login page par bhej diya jata hai

`db.sqlite3`

20. Images aur media kaise handle ho rahe hain
Settings me:
1. `STATIC_URL = 'static/'`
2. `MEDIA_URL = '/media/'`
3. `MEDIA_ROOT = BASE_DIR / 'media'`

Matlab:
Uploaded images `media` folder me save hoti hain.
Jaise products, projects, profile images.

Iska fayda:
